# 受け入れテスト項目（ashibaya-line）

本番URL: https://ashibaya-line.vercel.app

- ✅ = 期待どおり / ⚠️ = 要確認・不具合
- 「🧪切替」= リリース前限定の会社切り替えツール（右下）。使うには Vercel 環境変数 `NEXT_PUBLIC_ALLOW_ACTING_SWITCH=1` を設定。**テスト後は必ず外して再デプロイ**。

---

## 0. 事前準備
- [ ] `NEXT_PUBLIC_SUPABASE_URL` / `NEXT_PUBLIC_SUPABASE_ANON_KEY` / `SUPABASE_SERVICE_ROLE_KEY` が設定済み
- [ ] （テスト用）`NEXT_PUBLIC_ALLOW_ACTING_SWITCH=1` を設定して再デプロイ
- [ ] テスト用に会社を2社以上、本部管理者アカウント1つを用意

## 1. 認証・オンボーディング
- [ ] 未ログインで `/` を開くと公開ランディングが表示される（サンプルではなく案内）
- [ ] ブラウザのタブ名が各ページに沿っている（ホーム/案件/取引/自社 など）
- [ ] サインアップで会社登録できる（登録直後の状態は「承認待ち(pending)」）
- [ ] メール＋パスワードでログインできる → 一般会社は `/home` へ
- [ ] 管理者でログインすると `/admin` へ遷移する
- [ ] ログアウトできる
- [ ] 未ログインで `/home` 等を開くと `/login` に戻される

## 2. 本部承認（/admin）
- [ ] 会社一覧に登録会社が並び、状態バッジ（承認待ち/取引可/停止中）が出る
- [ ] 「承認」で active になる／「停止」で suspended になる
- [ ] 会社は自分で status を変更できない（管理者のみ）
- [ ] `SUPABASE_SERVICE_ROLE_KEY` 未設定でも /admin が 500 にならず赤い注意書きが出る

## 3. 承認ゲート（発注・受注は承認後）
- [ ] 承認待ちの会社はホームに「本部の承認待ちです」バナーが出る
- [ ] 承認待ちの会社は案件タブに「投稿は承認後」の案内、投稿ボタンが出ない
- [ ] 承認待ちの会社は案件応募（受注）ができない
- [ ] 承認(active)後、発注・受注が解禁される

## 4. 会社切り替え（テスト機能・フラグ有効時のみ）
- [ ] 右下に🧪ボタンが表示される（フラグ未設定時は出ない）
- [ ] 会社を選ぶと「選択中」になり、各画面がその会社として動作する
- [ ] 「切り替えを解除」でログイン会社に戻る

## 5. 案件投稿・編集（発注側）
- [ ] 「案件を投稿」→ 種別（応援/請負）・金額などを入力
- [ ] 金額欄で数字入力すると自動で3桁区切り（例: 22,000）
- [ ] 「入力内容を確認する」→ 確認モーダルに一覧表示 → 公開
- [ ] 投稿フォームに売掛保証トグルは無く「受注側が受注時に選択」の注記がある
- [ ] 一覧に投稿した案件が出る（自社案件バッジ、募集中バッジ）
- [ ] 案件詳細で「✎ 案件内容を編集する」が元請本人・募集中のみ表示
- [ ] 編集 → 確認モーダル → 更新が反映される
- [ ] 選定済み(matched)や他社の案件では編集ボタンが出ない

## 6. 応募（受注側）
- [ ] 🧪で別会社（承認済み）に切替 → 案件詳細で「この案件に応募する」
- [ ] 応募後、案件一覧にその案件で「応募中」バッジが出る
- [ ] 案件詳細に「応募中です。元請の選定をお待ちください。」が出る
- [ ] 自社案件には応募できない／二重応募できない

## 7. 応募の秘匿（プライバシー）
- [ ] 応募会社一覧は元請本人にのみ表示される
- [ ] 応募した会社は自分の応募状態のみ見え、他の応募者は見えない
- [ ] 無関係の第三者会社には応募者情報が一切見えない

## 8. 選定 → 取引生成
- [ ] 元請に切替 → 応募者一覧から「この会社に依頼」で取引が作成される
- [ ] 選定後、応募者側の表示が「選定は終了しました…」に変わる
      （＝“お待ちください”が出続けない）
- [ ] 取引タブに新しい取引が表示される

## 9. 通知欄（ホーム・案件タブ）
- [ ] 自社案件に応募が付くと元請に「応募」通知
- [ ] 選定されると受注側に「選定」通知（取引を開始してください）
- [ ] 相手が受注（取引開始）すると元請に「受注」通知（注文書を発行）
- [ ] 通知は次のアクションを行うと消える（滞留しない）

## 10. 取引エンジン（請負＝二相）
- [ ] 受注側で「取引を開始」→ **売掛保証トグル**が出て選択できる
- [ ] 取引情報の売掛保証が選択どおり反映される
- [ ] 元請「注文書を発行」→ 受注側「注文請書を発行」
- [ ] 組立：作業開始 → 完了報告 → 元請が完了確認（または是正依頼→是正完了）
- [ ] 解体は組立の完了確認後にのみ開始できる
- [ ] 請求（組立分/残金）→ 元請が確認 → 支払い → 受注側が入金確認
- [ ] 入金額が請求と異なると「確認事項」に登録される
- [ ] 両フェーズの入金確認で取引が「完了」になる

## 11. 取引エンジン（応援＝単相）★今回の変更
- [ ] 応援案件の取引は「作業を開始」のみで、組立/解体フェーズが出ない
- [ ] 募集要項・取引画面に「解体予定」等が出ず「作業予定/作業フェーズ」になる
- [ ] 作業 → 報告 → 確認 → 請求 → 入金確認 → 完了まで単相で進む

## 12. 信用タイムライン・信用実績
- [ ] 取引詳細に「信用タイムライン」が表示され、進行が時系列で積み上がる
- [ ] 取引開始・完了確認・入金確認が節目、取引完了が緑で強調される
- [ ] 取引完了後、相手会社の信用レベル・支払い実績（自社/登録会社一覧）が更新される

## 13. 案件チャット
- [ ] 案件詳細から、元請は各応募者ごと・受注側は元請とチャットを開ける
- [ ] チャットは依頼先（会社）ごとに分かれている
- [ ] 選定後（取引画面「チャットを開く」）も同じチャットが継続する
- [ ] 送信すると相手/自分の吹き出しで表示される（Cmd/Ctrl+Enterで送信）
- [ ] 関係者（元請・その依頼先）以外はチャットを開けない

## 14. 登録会社一覧
- [ ] ホームの登録会社一覧が各社クリックで開閉する
- [ ] 「信用レベル順 / 取引件数順」で並び替えできる
- [ ] 自社に「自社」バッジが付く

## 15. セキュリティ・リグレッション
- [ ] 無関係の第三者は他社の取引内容（金額・現場）を閲覧できない
- [ ] どのルートを開いても 500（MIDDLEWARE_INVOCATION_FAILED など）が出ない
- [ ] モバイル幅で各画面のレイアウトが崩れない

## 16. 後片付け
- [ ] テスト完了後、`NEXT_PUBLIC_ALLOW_ACTING_SWITCH` を削除して再デプロイ
- [ ] 🧪ボタンが消え、通常の（会社切り替え不可の）挙動に戻る
